# Divine Punishment — マイルストーン & Issue 分割方針

作成日: 2026-07-03
ステータス: Draft v1

## マイルストーン概要

各マイルストーンは「動くもの + 検証条件」で完了を定義する。順序依存を明示し、並行可能な範囲を広く取る。

| MS | 名称 | 完了条件（Exit Criteria） |
|---|---|---|
| M0 | プロジェクト基盤 | Godot プロジェクト・CI（GUT + エクスポート）・チューニングデータ枠組みが動く |
| M1 | 移動とカメラ | グレーボックス上で全スタンス移動・張り付き・登攀・床下・隠れ場所が動く |
| M2 | ステルス知覚 | 可視度・音・光源消灯が動き、デバッグ可視化で数値が読める |
| M3 | 敵 AI | 警戒 5 段階 FSM・巡回・捜索・異常検知が動き、鬼ごっこが成立する |
| M4 | 暗殺と戦闘 | 4 文脈の必殺・死体処理・剣戟・離脱 → 再潜伏が動き、コアループ 1 周が成立する |
| M5 | 忍具と標的 | 忍具 3 種・標的ルーチン・護衛が動く |
| M6 | ミッションとメタ | チュートリアル + 武家屋敷グレーボックスが通しでプレイでき、評価・セーブ・画面フローが完結する |
| M7 | アート & ポリッシュ | 武家屋敷のアート / 音響 / 演出が完成し、VS 受け入れ基準（要件定義 §4）を満たす |

## 依存関係

```
M0 → M1 → M2 → M3 → M4 → M6 → M7
              └──→ M5 ──↗
```

- M5（忍具・標的）は M2 完了後、M3/M4 と並行可能
- レベルグレーボックス制作（M6 の一部）は M1 完了後から先行着手可能
- アートアセット調達・検証（M7 の一部）は全期間並行可能

## Issue 分割方針

- 1 Issue = 1 PR 相当の縦切りスコープ（レビュー可能サイズ、目安 0.5〜3 日）
- 各 Issue に: 目的 / 対象ドキュメント参照 / 受け入れ条件（チェックリスト）/ 依存 Issue を記載
- ラベル: `milestone:M0`〜`M7` は GitHub Milestone で管理し、ラベルは領域（`area:player` `area:ai` `area:stealth` `area:level` `area:ui` `area:audio` `area:infra` `area:design`）+ `type:feature` `type:tuning` `type:test` を付与
- チューニング・プレイテスト系 Issue は「数値確定」を成果物とし、`data/tuning/` への反映と docs 更新を完了条件に含める
