# Divine Punishment — 要件定義書

作成日: 2026-07-03
改訂: 2026-07-03 v2 — スコープをフルゲーム（ストーリーエンドまで）に拡張（ユーザー承認済み）

スコープ: **フルゲーム（全 10 ミッション + ナラティブ + エンディング）**。開発は Phase 1（Vertical Slice、以下 VS）→ Phase 2（キャンペーン）の 2 段階で行うが、**完成の定義は Phase 2 完了**である。

## 1. スコープ定義

### 1.0 Phase 構成

| Phase | 範囲 | 対応 Milestone |
|---|---|---|
| Phase 1: VS | コアループ + M1/M2 + 評価システム（§1.1） | M0〜M7 |
| Phase 2: キャンペーン | M3〜M10 全ミッション、ナラティブ全編、拡張メカニクス、エンディング（§1.3） | M8〜M12 |

### 1.1 Phase 1（VS）に含むもの

| 領域 | 内容 |
|---|---|
| プレイヤー | 移動・しゃがみ・走り・壁張り付き・梁移動・床下潜入・水中潜行・よじ登り |
| ステルス | 可視度（光量ベース）・音伝播・敵の視覚 / 聴覚知覚・警戒段階 |
| 光と影 | 消灯可能な光源（灯篭・篝火）、敵による再点灯 |
| 暗殺 | 背後 / 頭上 / 床下 / 角からの一撃必殺、死体運搬・隠蔽 |
| 忍具 | 小石（誘導）・吹き矢（消灯 / 無力化）・煙玉（リカバリ）の 3 種 |
| 戦闘 | 発見時の簡素な剣戟（攻撃・防御・回避）と逃走・再潜伏 |
| 敵 AI | 巡回・視線・音反応・死体発見・警戒段階遷移・捜索・再点灯 |
| レベル | チュートリアル小ステージ 1 面 + 本番ミッション「武家屋敷」1 面 |
| 評価 | ミッションリザルト（未発見 / 死体未発見 / 標的一撃 / 時間 → ランク） |
| UI | タイトル・ミッション選択・HUD・ポーズ・リザルト |
| 音響 | 環境音・緊張度連動 BGM・足音（材質差）・SE |
| 保存 | ミッションクリア状況・ベストランク・設定 |
| 基盤 | Godot 4 プロジェクト構成・CI（headless テスト + ビルド）・PC ビルド |

### 1.2 プロジェクト全体に含まないもの（将来拡張）

- 2 人目の主人公・キャラ切替（アーキテクチャ上の考慮のみ行う）
- 成長要素・忍具ショップ・スコアアタックのオンラインランキング
- コンソール / モバイル対応、英語等ローカライズ（テキストは外部化のみ実施）
- ボイス収録・リアルタイム 3D カットシーン
- マルチエンディング分岐（単一エンド + エピローグ 2 変化のみ）

## 2. 機能要件

記法: **FR-カテゴリ-番号**。優先度は MoSCoW（M=Must / S=Should / C=Could）。

### 2.1 プレイヤー操作（PLR）

| ID | 要件 | 優先度 |
|---|---|---|
| FR-PLR-01 | 三人称カメラで移動（歩き / しゃがみ / 走り）ができ、スタンスで発する音量・可視度が変わる | M |
| FR-PLR-02 | 壁・物陰に張り付き、角から覗き見できる | M |
| FR-PLR-03 | 梁・屋根へよじ登り、頭上経路を移動できる | M |
| FR-PLR-04 | 床下に潜入し、床下経路を移動できる | M |
| FR-PLR-05 | 水中に潜り、息継ぎ制限つきで水中移動できる | S |
| FR-PLR-06 | 茂み・物陰・戸棚などの隠れ場所に入れる | M |
| FR-PLR-07 | ゲームパッドとキーボード / マウスの両対応、キーリマップ可能 | M（リマップは S） |

### 2.2 ステルスシステム（STL）

| ID | 要件 | 優先度 |
|---|---|---|
| FR-STL-01 | プレイヤーの可視度を周囲の光量・スタンス・移動速度・遮蔽から連続値で算出する | M |
| FR-STL-02 | 現在の可視度と発している音をプレイヤーが HUD で読める | M |
| FR-STL-03 | 足音は移動速度と床材質（畳 / 木板 / 砂利 / 水）で音量が変わり、敵の聴覚に届く | M |
| FR-STL-04 | 灯篭・篝火などの光源を忍具・接触操作で消灯できる | M |
| FR-STL-05 | 消灯された光源は、気付いた敵が一定時間後に再点灯する | M |
| FR-STL-06 | 小石などで音を発生させ、敵の注意を誘導できる | M |

### 2.3 敵 AI（AI）

| ID | 要件 | 優先度 |
|---|---|---|
| FR-AI-01 | 敵は巡回経路・立哨・休憩などのルーチンを持つ | M |
| FR-AI-02 | 視覚: 視野角・距離・遮蔽・対象の可視度に基づく発見メーター方式（即発見しない） | M |
| FR-AI-03 | 聴覚: 音イベントの音量と距離から反応し、音源を調べに行く | M |
| FR-AI-04 | 警戒段階: 平常 → 不審 → 捜索 → 戦闘 → 帰投 の状態遷移を持ち、段階が挙動・BGM・HUD に反映される | M |
| FR-AI-05 | 死体・消えた灯り・開いた戸などの異常を発見すると警戒が上がる | M |
| FR-AI-06 | 戦闘状態の敵はプレイヤーを攻撃し、見失うと捜索へ移行、一定時間で帰投する | M |
| FR-AI-07 | 敵同士の伝達（近くの敵への警戒伝播）を行う | S |

### 2.4 暗殺・戦闘（KIL）

| ID | 要件 | 優先度 |
|---|---|---|
| FR-KIL-01 | 未発見状態で背後・頭上（梁）・床下・角から一撃必殺を実行できる（文脈依存の短い演出付き） | M |
| FR-KIL-02 | 必殺の成立条件（距離・角度・敵の警戒段階）が一貫したルールで判定される | M |
| FR-KIL-03 | 死体を担いで移動し、茂み・床下・水中に隠せる | M |
| FR-KIL-04 | 発見時は剣戟（攻撃・防御・回避）が可能だが、複数敵相手には明確に不利 | M |
| FR-KIL-05 | 煙玉などで戦闘から離脱し、隠れ直すことで警戒を解除できる | M |
| FR-KIL-06 | プレイヤーの被弾 → 死亡 → チェックポイント / リトライが機能する | M |

## 3. 非機能要件

| ID | 要件 |
|---|---|
| NFR-01 | 対象: Windows / macOS / Linux（Godot 4 エクスポート）。ミドルレンジ GPU で 1080p / 60fps 目標 |
| NFR-02 | 知覚判定（光・音・視線）は毎フレーム全数計算せず、間引き・距離カリングで敵 12 体時も 60fps を維持 |
| NFR-03 | 公式 IP の名称・キャラ・ロゴ・音源・アセットを一切含まない（権利クリーン）。アセットはライセンス確認済みのもののみ |
| NFR-04 | ステルスルールの数値（視野角・光量閾値・音半径など）はデータファイルで調整可能（デザイナーチューニング可能性） |
| NFR-05 | 知覚計算・評価集計はユニットテストを持ち、CI（GitHub Actions, headless Godot）で自動実行 |
| NFR-06 | セーブデータはユーザーディレクトリに JSON / Resource で保存。破損時は初期化 |
| NFR-07 | キャラ切替拡張に備え、プレイヤー能力値（移動音・速度・可搬忍具）はプロファイルデータとして分離 |

## 4. 受け入れ基準

### 4.1 Phase 1（VS）完了の定義

1. 新規プレイヤーがチュートリアルを経て「武家屋敷」を未発見クリアでき、上位ランクが取れる
2. 発見された場合も、逃走 → 再潜伏 → 攻略続行が成立する
3. 意図しない発見（理不尽検知 / 検知漏れ）のバグ報告ゼロ状態でプレイテスト 1 巡通過
4. CI green（テスト + 3 OS ビルド成功）
5. 全ドキュメント（本書・設計書群）が実装と一致するよう更新済み

### 4.2 Phase 2（完成 = プロジェクト完了）の定義

1. OP → M1〜M10 → 終幕 → エピローグまでノーバグで通しプレイ可能（想定 6〜8 時間）
2. 全 10 ミッションで経路 3 系統それぞれの未発見クリアが可能（07 付録チェックリスト全通過）
3. M9 の殺害禁止ルール・M10 の決闘が仕様通りに成立する
4. 修羅値による台詞分岐・エピローグ 2 変化が正しく発火する（両パターンの通し確認）
5. カットシーン 23 本・隠れ家 8 回・静止画約 70 枚・BGM/SE が全て組み込まれ、テキストは 06-narrative.md の確定台詞と一致する
6. 外部プレイテスター 2 名以上がエンディング到達し、致命 / 進行不能バグゼロ
7. CI green + 3 OS のリリースビルドが起動から終幕まで動作
8. 全ドキュメントが最終実装と同期済み・アセットライセンス台帳完備
